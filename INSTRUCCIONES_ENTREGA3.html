2 Guardar imagen como base64 en forms/form-fill/form-fill.page.ts código

El método onFileChange actualmente solo guarda el nombre del archivo. Reemplazarlo con la lectura real en base64:

onFileChange(event: Event, fieldName: string): void {
  const input = event.target as HTMLInputElement;
  if (!input.files?.length) return;

  const reader = new FileReader();
  reader.onload = () => {
    // Guarda el base64 completo — se persiste en localStorage
    this.formData[fieldName] = reader.result as string;
  };
  reader.readAsDataURL(input.files[0]);
}

Este base64 ya queda guardado automáticamente cuando el piloto guarda borrador o envía el formulario.

🔄 RF pendiente — Sincronización offline → online
3 Implementar shared/services/sync.service.ts código

El método syncPending() está vacío. Implementarlo para que marque los formularios enviados como sincronizados:

async syncPending(): Promise<void> {
  if (!this.isOnline()) return;

  const entries = this.storage.get<any[]>('form_entries') ?? [];
  const pendientes = entries.filter(e => e.status === 'submitted').length;
  if (pendientes === 0) return;

  const actualizados = entries.map(e =>
    e.status === 'submitted'
      ? { ...e, status: 'synced', updatedAt: new Date().toISOString() }
      : e
  );
  this.storage.set('form_entries', actualizados);
}

getPendingCount(): number {
  const entries = this.storage.get<any[]>('form_entries') ?? [];
  return entries.filter(e => e.status === 'submitted').length;
}
4 Detectar reconexión en app/app.component.ts código

Inyectar SyncService y ToastController en el AppComponent y escuchar el evento online del navegador:

// En el constructor, agregar:
constructor(
  private syncService: SyncService,
  private toastCtrl: ToastController
) {}

ngOnInit(): void {
  window.addEventListener('online', () => this.onConexionRecuperada());
}

async onConexionRecuperada(): Promise<void> {
  const pendientes = this.syncService.getPendingCount();
  if (pendientes === 0) return;

  const toast = await this.toastCtrl.create({
    message: `Conexión restaurada. ${pendientes} formulario(s) pendientes.`,
    position: 'top',
    color: 'success',
    buttons: [{
      text: 'Sincronizar',
      handler: () => this.syncService.syncPending()
    }],
    duration: 8000
  });
  await toast.present();
}

Recuerda importar ToastController de @ionic/angular/standalone y SyncService del servicio.

5 Sección: Publicación en Google Play Store documento
  • Cuenta Google Play Developer: costo único de USD $25
  • Proceso: crear ficha de aplicación → subir APK/AAB → revisión de Google (3-7 días hábiles)
  • Requisitos: política de privacidad, capturas de pantalla, descripción en español
  • Distribución: se puede restringir a empresas (distribución interna por MDM) o publicar abierta
6 Sección: Estrategias de monetización para ANKA documento

ANKA RAC100 es una app de nicho B2B (empresas operadoras de drones). Analizar al menos 3 modelos:

  • Modelo 1 — Licencia empresarial: precio anual por empresa (ej. COP $2.500.000/año), incluye todos los formularios, usuarios ilimitados y soporte. Ideal para operadoras de aerofotos e inspección.
  • Modelo 2 — Freemium: versión gratuita con 1 formulario activo y 2 pilotos. Versión Pro (COP $80.000/mes) con formularios ilimitados, sincronización cloud y exportación PDF.
  • Modelo 3 — Integración con sistemas externos: cobrar por API de sincronización con ERPs de empresas (SAP, etc.) o con plataformas de aerofotografía (Pix4D, DroneDeploy).
  • Recomendación: para el contexto colombiano, el modelo de licencia empresarial anual es el más viable dado el nicho regulado (RAC 100 de la Aeronáutica Civil).
